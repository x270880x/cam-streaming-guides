簡単な答え
複数のcamサイトに一度に配信するには：SplitCamをインストール、シーンを構築、各camサイトのRTMPサーバーURLとストリームキーを取得、すべてをSplitCamマルチストリーミング設定に追加、Go Liveを一度クリック。

ビデオガイド

ステップバイステップ

1
SplitCamをインストール

SplitCamはマルチストリーミング内蔵のWindowsとmacOS用の無料配信ソフトウェア。インストール — 透かしなし、登録なし。

2
カメラとシーンをセットアップ

SplitCamを開いてウェブカメラを追加し、オーバーレイとフィルターでシーンを構築。1つのシーンが各宛先に供給。

3
各camサイトからRTMPキーを取得

各camプラットフォームで外部 / RTMP配信を有効化し、サーバーURLとストリームキーをコピー。配信したい各サイトで繰り返し — 正確なルートは個別のプラットフォームガイドを参照。

4
各宛先をSplitCamに追加

Stream Settingsを開いて各camサイトをカスタムRTMP宛先として追加 — サーバーURLとキーを貼り付け。ライブにしたいすべてをチェック。

5
Go Liveを一度クリック

Go Liveを押す。SplitCamはストリームを各選択されたcamサイトに同時に、ピアツーピアで、単一のエンコーディングから送信 — 追加料金なし。

プロのヒント

アップロードを監視

マルチストリーミングはアップロード負荷を倍増。各宛先は独自のビットレートを消費 — 接続が合計を処理できることを確認。

プラットフォームルールを確認

一部のcamサイトは他の場所での同時配信を禁止 — マルチストリーミング前に確認。

ケーブル — ここでドロップは許されない

マルチストリーミングはアップロード負荷を倍増、wi-fiドロップ1つですべての宛先を一度に倒す可能性。ケーブルはここでオプションではない。

ヘルスモニターを監視

SplitCamは宛先ごとにステータスを表示 — アップロードが対応できない場合はサイトをドロップ。

トラブルシューティング

Multiple Cam Sites の配信がカクつく・バッファリングする

ほぼ常にビットレートがアップロードの限界を超えています。SplitCam の内蔵速度テストを実行し、ビットレートを測定したアップロードの約75%に設定してください — 1080pで3,500–6,000 Kbps、720pはそれ以下。エンコーダーが接続を追い越さなくなればカクつきは消えます。

Multiple Cam Sites 配信中のフレーム落ち

フレーム落ちはパケットが Multiple Cam Sites に間に合っていないこと — 通常は不安定なWi-Fiが原因です。有線Ethernet接続に切り替え、帯域を食うアプリを閉じ、ビットレートを少し下げてください。単発のスパイクは問題なし；継続的な増加は接続が追いつかないサインです。

黒い画面 — Multiple Cam Sites で視聴者に映像が映らない

カメラが SplitCam でアクティブソースとして選択されていないか、別のアプリが使用中です。Zoom、Skype、OBS を閉じ、SplitCam のソースリストでウェブカメラを選び直し、Go Live を押す前にプレビューに映像が表示されることを確認してください。

Multiple Cam Sites がストリームキーを拒否する・接続しない

ストリームキーをコピーし直してください — 末尾のスペースや古い、すでにローテートされたキーが通常の原因です。サーバーURLが Multiple Cam Sites の表示と一致し、アカウントで外部エンコーダー配信が有効か確認してください。SplitCam の Stream Settings の緑のスライダーが有効なキーを確認します。

Multiple Cam Sites で音声がない・音ズレする

SplitCam をカメラとマイクの両方に選び、SplitCam の音声ソースで実際のマイクを選択してください。音声が映像から遅れる場合は解像度を1段下げてください — エンコーダーが過負荷で、音声が遅れたフレームを待っています。

よくある質問

SplitCamマルチストリーミングは無料？

はい — マルチストリーミングは内蔵で無料、宛先ごとの料金なし、透かしなし。

同時に何個のcamサイトに配信できる？

アップロード帯域幅が処理できる数 — 各宛先は独自のビットレートを消費。

クラウドリレーを使用？

いいえ — SplitCamはストリームをピアツーピアでPCから各プラットフォームのingestに直接送信。

マルチストリーミングはPCを遅くする？

エンコーディングは一度行われ再利用；ハードウェアエンコーディングはCPU負荷を低く保つ。アップロード帯域幅が実際の制限。
